## Quick Answer

In the observed NOUN portal session, the `Grade` column existed on `My progress`, but the visible rows showed dashes. The safest response is not to assume the final reason. First confirm the page is loading the expected academic periods, then open the Support portal and compare the deeper ERP result statement. If the grade still does not resolve there, save evidence and use the official support route.

## What Was Actually Observed

The live session confirmed three facts that matter for this issue:

1. `My progress` is the first results page
2. the page includes a `Grade` column
3. visible rows on this account displayed `-`

What the session did not confirm:

- whether the dash meant pending upload
- whether it meant a missing record
- whether it meant a faculty-specific delay

That boundary matters. This guide helps you investigate the problem using verified surfaces. It does not invent reasons the system did not show.

## Why This Problem Feels Bigger Than It Is

When students see a dash in the grade area, they often jump straight to major conclusions:

- my result is gone
- the portal failed
- I need to register again immediately
- I have failed and the portal has not updated properly

The verified workflow suggests a calmer approach. One page showed partial information, but the system exposed a second, deeper result path through Support. So the job is to compare surfaces before you interpret the outcome.

## Step 1: Confirm You Are On My Progress

Make sure you are looking at the correct first result screen. The verified entry point was `My progress`, not a menu explicitly called `Results`.

Check whether the page already shows:

- course rows
- academic periods such as `2024_1` or `2024_2`
- the `Grade` column
- cumulative passed-credit summaries

If these elements are present, then the academic-history page is loading and you are in the right place to begin.

## Step 2: Note What The Grade Area Actually Shows

Do not summarize the issue vaguely as `my result is not showing`. Be precise about what you see.

In the observed session, the correct description would have been:

- the `Grade` column is visible
- the current visible rows show `-`

That is a better starting point than saying the entire result page is missing. It tells you the system is returning structure, but not the final value you expected.

## Step 3: Move To The Deeper Result Surface

The next step is essential. The `My progress` page itself instructed students to use `Support` and `eTicketing` to see more results.

So the verified comparison workflow is:

1. open `My progress`
2. note the dash or incomplete grade state
3. go to `Support`
4. log into the Support portal
5. click `Result statement`
6. review the ERP grade table

This is the correct way to test whether the problem is only on the first page or whether it still appears in the deeper record.

## Step 4: Compare Both Result Surfaces

Use this comparison checklist:

- does `My progress` show the expected periods?
- does the ERP statement page load a detailed course table?
- do the course codes line up with what you expect?
- does the deeper page show the same incomplete state or more detail?
- are batches such as `2024_1` and `2024_2` visible?

This comparison matters because it turns a vague complaint into a traceable result problem.

## Step 5: Save The Right Evidence

Before you escalate, collect safe evidence:

- a safe screenshot of the `My progress` page state
- a safe screenshot of the ERP grade table or relevant section
- the course code
- the period or batch visible on the page
- the exact wording of the issue you saw

This is much stronger than writing a message that only says `my grade is not showing`.

## What Not To Assume

Because the session did not verify the underlying reason for the dash, avoid unsupported claims like:

- the result has been withheld
- the portal is waiting for a correction request
- the system is showing a carryover label somewhere hidden
- the grade must appear after a fixed number of days

None of those were directly confirmed in the research phase.

## Common Mistakes To Avoid

- confusing `Grade column exists` with `full grade is confirmed`
- stopping at My progress and never checking the ERP result page
- describing the issue too broadly when support needs page-specific evidence
- mixing up different academic periods
- using an old screenshot as if it proves the current record

## Troubleshooting

### The grade column is there, but all I see is a dash

That is exactly the verified problem pattern. Move to the Support-portal result statement and compare the deeper table.

### I can see some history, but not enough to trust the result

Use the ERP statement page as the second verification layer. That page exposed fuller academic detail in the observed session.

### I still do not understand whether the issue is a result problem or a registration problem
