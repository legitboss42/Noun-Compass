## Quick Answer

To submit a TMA on NOUN eLearn, first open the correct TMA activity page, verify the `Opened` and `Closed` timestamps, confirm the number of allowed attempts, and only then proceed through the official `Attempt TMA Now` path. After the work is completed, check the attempt summary for `Status`, `Started`, `Completed`, and `Duration`, then verify the score through `Grades`.

The critical lesson from the observed session was simple: do not treat the TMA page as a place for rushed clicking. Read the page first.

## Important Scope Note

This guide is based on a real observed eLearn workflow, but the live session did not expose a fresh first-time editable TMA screen because the visible TMAs had already been completed. That means this article documents:

- the real pre-action checks shown on the TMA page
- the observed instruction label `Attempt TMA Now`
- the visible attempt restriction
- the real finished-state summary after submission
- the real grade-report path after submission

It does not invent button-by-button steps for an unobserved form.

## The Submission Workflow We Can Confirm

Based on the live session, the reliable workflow is:

1. find the correct course and TMA
2. open the TMA activity page
3. read the timing and attempt rules
4. use the official attempt path only when ready
5. after finishing, verify the attempt summary
6. check the grade surfaces later

This is not generic advice. It matches the exact page structure observed in eLearn.

## What To Do Before Clicking Attempt TMA Now

### Step 1: Confirm the course and TMA label

Make sure the visible TMA belongs to the correct course. In the observed session, the course was `Numerical Analysis (MTH213_261)` and the TMA labels were clearly separated into `TMA1`, `TMA2`, and `TMA3`.

### Step 2: Read the `Opened` and `Closed` fields

Do not assume the assignment is active because you can see the page. The activity header explicitly displayed timing.

### Step 3: Check `Attempts allowed`

The observed TMAs displayed `Attempts allowed: 1`. That should change your behavior immediately. One-attempt tasks should never be opened casually.

### Step 4: Prepare your environment

Before using the official attempt path, make sure you are on a stable connection, in the correct account, and free from distractions. This is especially important when the observed workflow shows only one chance.

### Step 5: Save the right evidence after completion

Once the task is done, the post-submission state matters. You want proof that the system recorded the attempt.

## How To Confirm A TMA Really Went Through

After submission, use this checklist:

1. open the same TMA activity page again
2. look for the attempt summary card
3. confirm `Status Finished`
4. confirm the recorded `Started` and `Completed` times
5. note the `Duration`
6. check whether the page says `Review not permitted`
7. move to `Grades` to verify scoring later

This is a safer pattern than depending on memory or assuming the LMS saved everything because the screen changed briefly.

- the broader `Grades` overview

## Common Submission Mistakes To Avoid

- opening the TMA without first reading `Opened`, `Closed`, and `Attempts allowed`
- assuming the task permits several tries
- confusing the TMA page with the grade-report page
- failing to verify the finished attempt summary after the task
- rushing because the dashboard shortcut made the TMA easy to open
- asking for support without a course code, TMA name, or safe screenshot

These mistakes are avoidable because the observed platform already exposes the signals students need. The issue is usually attention, not a lack of labels.

## Troubleshooting

### The TMA page is visible but I am not ready to start

Do not click through just because you found the page. Read the timing and attempt rule, then return when you are prepared.

### The page still says Attempt TMA Now but also looks completed

In the observed session, the instruction line remained visible while the attempt summary below clearly showed `Finished`. Read the whole page, not only the top instruction.

### I finished the TMA but I cannot tell whether it saved

Check the attempt summary for `Status Finished`, `Started`, `Completed`, and `Duration`. That was the strongest post-submission proof in the observed workflow.

### I need to know whether the score has appeared

Move out of the TMA page and check `Grades`, especially the course-specific grade report.

### I still need official help

Prepare a safe screenshot of the TMA state, note the course code and TMA label, then use the observed eLearn support form or the [NOUN Support Ticket Guide](/articles/noun-support-ticket-guide).
